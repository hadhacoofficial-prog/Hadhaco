# Hadha.co — GitHub Secrets Configuration Guide

## Overview

All secrets are injected via GitHub Secrets and passed to the VPS at deploy time.
**Never hardcode credentials in the repository or on the server.**

---

## Required Secrets

### Server Access

| Secret | Description | How to Generate |
|--------|-------------|-----------------|
| `SSH_HOST` | VPS IP address | Your server IP |
| `SSH_PORT` | SSH port (default: 22) | Usually `22` |
| `SSH_USER` | SSH username | `deploy` |
| `SSH_PRIVATE_KEY` | SSH private key for deploy user | Generated by `bootstrap.sh` — printed on first run |

### Docker Registry

| Secret | Description | How to Generate |
|--------|-------------|-----------------|
| `GHCR_TOKEN` | GitHub PAT with `read:packages` scope | GitHub → Settings → Developer Settings → Personal access tokens → Fine-grained → `read:packages` on `hadhacoofficial-prog/Hadhaco` |

### Application Backend

| Secret | Description | How to Generate |
|--------|-------------|-----------------|
| `SECRET_KEY` | Application secret key | `python -c "import secrets; print(secrets.token_hex(32))"` |
| `ENCRYPTION_KEY` | Fernet encryption key | `python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"` |
| `DATABASE_URL` | Supabase session pooler URL | Supabase Dashboard → Settings → Database → Connection string → Session mode |
| `ALEMBIC_DATABASE_URL` | Supabase direct connection URL | Supabase Dashboard → Settings → Database → Connection string → Direct connection |
| `SUPABASE_URL` | Supabase project URL | Supabase Dashboard → Settings → API → Project URL |
| `SUPABASE_ANON_KEY` | Supabase publishable/anon key | Supabase Dashboard → Settings → API → anon public |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key | Supabase Dashboard → Settings → API → service_role (secret) |

### Redis

| Secret | Description | How to Generate |
|--------|-------------|-----------------|
| `REDIS_PASSWORD` | Redis authentication password | `openssl rand -hex 32` |
| `REDIS_UI_USERNAME` | Redis Commander HTTP Basic Auth username | Choose a username (e.g., `hadha-admin`) |
| `REDIS_UI_PASSWORD` | Redis Commander HTTP Basic Auth password | `openssl rand -hex 16` |

### Cloudflare R2

| Secret | Description | How to Generate |
|--------|-------------|-----------------|
| `CLOUDFLARE_ACCOUNT_ID` | Cloudflare account ID | Cloudflare Dashboard → Right sidebar |
| `CLOUDFLARE_R2_ACCESS_KEY` | R2 API access key | Cloudflare → R2 → Manage R2 API Tokens → Create Token |
| `CLOUDFLARE_R2_SECRET_KEY` | R2 API secret key | Shown once at token creation |

### Email (Resend)

| Secret | Description | How to Generate |
|--------|-------------|-----------------|
| `RESEND_API_KEY` | Resend API key | Resend Dashboard → API Keys → Create |
| `RESEND_FROM_EMAIL` | Notification sender email | e.g., `notifications@hadha.co` |
| `RESEND_TO_EMAIL` | Notification recipient email | e.g., `admin@hadha.co` |

### Error Tracking (GlitchTip)

| Secret | Description | How to Generate |
|--------|-------------|-----------------|
| `GLITCHTIP_SECRET_KEY` | GlitchTip session signing key | `openssl rand -hex 32` |
| `GLITCHTIP_DB_PASSWORD` | GlitchTip PostgreSQL password | `openssl rand -hex 32` |
| `GLITCHTIP_DSN` | Backend error tracking DSN | Set after GlitchTip is running: `https://<key>@errors.hadha.co/<project_id>` |
| `GLITCHTIP_FRONTEND_DSN` | Frontend error tracking DSN | Set after GlitchTip is running: separate project DSN |

### Monitoring

| Secret | Description | How to Generate |
|--------|-------------|-----------------|
| `GRAFANA_USERNAME` | Grafana admin username | Choose a username (e.g., `admin`) |
| `GRAFANA_PASSWORD` | Grafana admin password | `openssl rand -hex 16` |

| `DOZZLE_USERNAME` | Dozzle monitoring dashboard username | Choose a username (e.g., `hadha-admin`) |

### Payments (Razorpay)

## Setup Procedure

### 1. Run Bootstrap

```bash
# On a fresh Ubuntu 24.04 VPS:
curl -fsSL <bootstrap-url> | sudo bash
```

Bootstrap will output:
- SSH private key (copy to `SSH_PRIVATE_KEY` secret)
- Generated passwords (copy to corresponding secrets)

### 2. Configure GitHub Secrets

Go to: `https://github.com/hadhacoofficial-prog/Hadhaco/settings/secrets/actions`

Add all secrets from the tables above.

### 3. Configure DNS

Add A records for all subdomains:

```
hadha.co          → <SERVER_IP>
www.hadha.co      → <SERVER_IP>
api.hadha.co      → <SERVER_IP>
admin.hadha.co    → <SERVER_IP>
redis.hadha.co    → <SERVER_IP>
dozzle.hadha.co   → <SERVER_IP>
grafana.hadha.co  → <SERVER_IP>
prometheus.hadha.co → <SERVER_IP>
cadvisor.hadha.co → <SERVER_IP>
uptime.hadha.co   → <SERVER_IP>
errors.hadha.co   → <SERVER_IP>
cdn.hadha.co      → <CLOUDFLARE_R2_ENDPOINT>
```

### 4. Deploy

Push to `main` branch to trigger the deployment workflow.

## Secret Rotation

### Redis Password

1. Generate new password: `openssl rand -hex 32`
2. Update `REDIS_PASSWORD` GitHub Secret
3. Redeploy (the deploy script will update Redis config)

### Monitoring Credentials

1. Generate new passwords
2. Update `MONITORING_PASSWORD`, `REDIS_UI_PASSWORD`, `DOZZLE_PASSWORD` GitHub Secrets
3. Redeploy

### SSL Certificates

Certificates auto-renew via cron. If renewal fails:
```bash
sudo certbot renew --force-renewal
docker exec hadha-nginx nginx -s reload
```

## Security Best Practices

- **Never** commit secrets to Git
- **Never** store secrets in plain text on the VPS (deploy script generates hashed versions)
- **Rotate** secrets periodically (every 90 days recommended)
- **Use fine-grained tokens** for GHCR access (read:packages only)
- **Monitor** GlitchTip for unauthorized access attempts
- **Review** Grafana audit logs periodically
